# CLAUDE.md — dation-pos

Макет **POS-терминала** «Dation» (React + Vite). Рабочее место администратора/мастера в салоне.
Отдельный проект от админки (`dation-mockup`) и клиентского Mini App (`dation-miniapp`).

Дизайн: фиолетовый бренд `#7C3AED`, **без эмодзи**, крупные тач-цели (терминал/планшет),
полноэкранный режим, поддержка светлой/тёмной темы.

## Состав (намеренно немного экранов)
1. **Авторизация** — вход терминала по номеру + паролю (`screens/Login.jsx`).
2. **Сотрудники** — профили тех, у кого есть доступ к POS; вход под своим профилем по PIN (`screens/StaffSelect.jsx`).
3. **Журнал записей** — подтягивается из веб-аппа; полный флоу: создание, управление, оплата (`screens/Journal.jsx`).
4. **Настройки** + вкладка **Интеграции** (`screens/Settings.jsx`).

Поток: Авторизация → выбор профиля сотрудника → Журнал (рабочий экран). В шапке — смена профиля и выход.

## Главное правило: удаляем ПОЛНОСТЬЮ
Когда из макета убирается элемент/функционал — удалять и из UI, и из кода фронтенда.
Не прятать, не комментировать. Убирать связанные мок-данные, состояния, обработчики, импорты.
После каждого изменения добиваться **0 предупреждений**: `npx eslint src`.

## Команды
- `npm run dev` — дев-сервер (порт 5190)
- `npm run build` — прод-сборка (проверка компиляции)
- `npx eslint src` — поиск мёртвого кода
