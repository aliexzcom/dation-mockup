# CLAUDE.md — dation-mockup

Макет веб-приложения «Dation» (React + Vite) по ТЗ. Дизайн: фиолетовые кнопки `#7C3AED`, белый фон, без эмодзи.

## Главное правило: удаляем ПОЛНОСТЬЮ

Когда из макета убирается любой элемент/функционал — удалять его и из UI, и из кода фронтенда. Не прятать, не комментировать, не оставлять «на потом».

При каждом удалении убирать всё связанное:
- разметку (JSX);
- мок-данные (массивы/константы), которые отображались только в этом блоке;
- состояния (`useState`), обработчики и функции, которые больше нигде не используются;
- ставшие лишними импорты (компоненты, иконки).

После удаления проверять проект на мёртвый код и добиваться **0 предупреждений**:

```
npx eslint src
```

(ESLint настроен: `eslint.config.js`, правило `unused-imports/no-unused-imports` + `unused-imports/no-unused-vars`.)

## Полезные команды
- `npm run dev` — дев-сервер
- `npm run build` — прод-сборка (быстрая проверка, что всё компилируется)
- `npx eslint src` — поиск неиспользуемого кода
