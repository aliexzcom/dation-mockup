# CLAUDE.md — dation-miniapp

Макет **Telegram Mini App** клиентского контура «Dation» (React + Vite) по ТЗ, п. 5.
Это клиентское приложение записи (внутри Telegram), отдельное от админки `dation-mockup`.

Дизайн: фиолетовый бренд `#7C3AED`, белый фон, **без эмодзи**, мобильный формат
(рамка «телефона»), поддержка light/dark темы Telegram.

## Флоу записи (7 экранов, ТЗ 5.1)
Филиал → Услуги (мультивыбор) → Мастер → Дата и время → Контакты → Депозит → Успех.
Плюс личный кабинет «Мои записи» (будущие/прошлые) — `screens/MyBookings.jsx`.

## Главное правило: удаляем ПОЛНОСТЬЮ
Когда из макета убирается элемент/функционал — удалять и из UI, и из кода фронтенда.
Не прятать, не комментировать. Убирать связанные мок-данные, состояния, обработчики, импорты.
После каждого изменения добиваться **0 предупреждений**: `npx eslint src`.

## Команды
- `npm run dev` — дев-сервер (порт 5180)
- `npm run build` — прод-сборка (проверка компиляции)
- `npx eslint src` — поиск мёртвого кода
